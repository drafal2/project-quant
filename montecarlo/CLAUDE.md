# montecarlo/

Foundation for the Monte Carlo engine that will price equity basket autocalls. This package implements the random-number sampling layer, the diffusion-side volatility models, and the single-asset path engine. Variance reduction (antithetic, Brownian bridge) and the multi-asset basket extension arrive in subsequent PRs of the path-engine roadmap.

## Layout

- **`sampler.py`** тАФ `Sampler` ABC. The single non-negotiable contract is `next_block(n_paths, n_dimensions) -> np.ndarray` returning a `(n_paths, n_dimensions)` `float64` array with values strictly in `(0, 1)`. The class attribute `is_quasi: bool` flags low-discrepancy samplers.
- **`uniform/`** тАФ six concrete `Sampler` implementations. PRNGs: `KnuthSampler` (Knuth RANARRAY lagged-Fibonacci in IEEE doubles, KK=100/LL=37/TT=70 тАФ **bit-for-bit compatible with `ql.KnuthUniformRng`** for any non-zero seed), `LecuyerMRG32k3a1999Sampler` (L'Ecuyer 1999 combined MRG, period ~2^191, production default for serial paths; `substream(index)` stub raises `NotImplementedError` until the path-engine PR), `LecuyerLCG1988Sampler` (L'Ecuyer 1988 combined-LCG + Bays-Durham shuffle, period ~2^61, **bit-for-bit compatible with `ql.LecuyerUniformRng`** тАФ included only for cross-validation; use `LecuyerMRG32k3a1999Sampler` in production), `MersenneTwisterSampler` (canonical MT19937 with vectorised tempering and a Python-loop refill, bit-for-bit compatible with `ql.MersenneTwisterUniformRng`; also exposes `next_int32_block(n)` for raw tempered uint32 output). QMC: `HaltonSampler` (prime-base van der Corput; supports both the deterministic origin-skip mode and a `random_start=True` mode that draws per-dimension start indices from MT19937 тАФ the latter is bit-for-bit compatible with `ql.HaltonRsg(d, seed, randomStart=True)` for any non-zero seed), `SobolSampler` (gray-code Sobol with Joe-Kuo (2008) direction numbers covering dimensions 1-1024, table in `_joe_kuo_data.py`, matches `ql.SobolRsg(JoeKuoD6)` up to a constant 2^-33 open-interval ULP shift).
- **`normal/`** тАФ `NormalTransform` ABC + five concrete transforms + `make_normal_sampler` factory. The factory enforces the **QMC / inversion pairing rule** (see Invariants below). Inversion family: `MoroTransform` (1995, ~3e-9 in the body, degrades in the deep tail), `AcklamTransform` (2003, ~1e-8 across the full range), `WichuraAS241Transform` (1988, machine precision throughout тАФ the default for tail-sensitive products). Trigonometric: `BoxMullerTransform` (PRNG-only). Pedagogy-only: `CLTTransform` (sum of 12 uniforms тИТ 6; issues `UserWarning` on construction and is rejected by the factory when paired with QMC).
- **`diagnostics/`** тАФ `uniform_tests.py` (KS, chi-square, serial correlation, Warnock L2 discrepancy, pairwise correlation grid), `normal_tests.py` (moments, KS vs `N(0, 1)`, Anderson-Darling, tail-fraction empirical vs theoretical), `integration.py` (`integrate_gaussian_moment` and `bs_call_price_mc` тАФ end-to-end smoke tests against closed-form benchmarks).
- **`plotting.py`** тАФ `scatter_2d`, `lag_scatter`, `projection_grid` (the canonical Halton-failure visual), `qq_normal`, `marginal_histogram`, `convergence_plot`. Every function takes an optional `ax: matplotlib.axes.Axes` and returns it; library code never calls `plt.show()`.
- **`volatility/`** тАФ diffusion-side volatility models consumed by the path engine. `VolModel` ABC with `diffusion(time, spot, state=None) -> np.ndarray`; `ConstantVol(sigma)`; `BlackTermStructureVol(surface)` which reads ATM total variance from a `market_structures.volatility.VolSurface` and returns a piecewise-constant instantaneous vol that reprices the surface's pillar ATM vanillas exactly under MC (skew ignored); `DupireLocalVol(surface)` тАФ the **full skew-aware bridge**. Dispatches on `isinstance(surface, DifferentiableVolSurface)`: for parametric surfaces (SVI, SSVI) it evaluates the Gatheral form of Dupire's formula analytically on every `diffusion()` call; for plain `VolSurface` (e.g. `InterpolatedVolSurface`) it pre-computes ``sigma_loc^2`` on an auto-sized (~50├Ч100) ``(t, k_log)`` grid via central finite differences and bilinearly interpolates at runtime. Clips to a positive ``sigma_floor`` (default ``0.01``) where the Dupire denominator collapses (butterfly arbitrage) or the numerator goes negative (calendar arbitrage); emits `UserWarning` if more than 1% of grid points are clipped. The `state` kwarg is plumbed from day one for future stochastic-vol models (Heston, SABR) that will read `state["variance"]` / `state["alpha"]`; the three current models ignore it.
- **`paths/`** тАФ Monte Carlo path generation. `TimeGrid` is the year-fraction grid `[0 = t_0 < t_1 < ... < t_N]` with `from_year_fractions` and `from_dates` (ACT/365) constructors. `PathEngine` ABC returns the `(n_paths, n_steps + 1, n_assets)` spot tensor. `EulerLogPathEngine` is the concrete single-/multi-asset engine: integrates the **log-spread** `X_t = log(S_t / F(t))` rather than spot, which absorbs the drift entirely into the forward callable (`-0.5 sigma^2 dt + sigma sqrt(dt) Z`, no remaining drift term). `sigma_k` is read at the **start** of each step from `vol_model.diffusion(t_k, S_k)`; `t = 0` is bumped to `1e-12` to honour the `DupireLocalVol` strict-positive contract without special-casing. The engine asks `normal_sampler` for one block of shape `(n_paths, n_steps * n_assets)` per `simulate()` call, preserving the dimension structure that PR 3 (Brownian bridge) and PR 4 (correlation) will reorder. `antithetic`, `brownian_bridge`, and `correlation` kwargs are accepted but inert in this PR тАФ they raise `NotImplementedError` if set to a non-default value, with messages naming the future PR. See [`paths/CLAUDE.md`](paths/CLAUDE.md) for the full step derivation.

## Invariants

- **QMC / inversion pairing rule** тАФ `is_quasi=True` samplers (`SobolSampler`, `HaltonSampler`) must be paired with `qmc_safe=True` transforms (`MoroTransform`, `AcklamTransform`, `WichuraAS241Transform`). Box-Muller wraps two uniforms through `(cos 2╧АU, sin 2╧АU)` non-monotonically, which destroys low-discrepancy structure (Boyle, Broadie, Glasserman 1997). The CLT transform is also non-monotone and tail-truncated. `make_normal_sampler` raises `ValueError` on bad pairings тАФ never bypass the factory.
- **Open interval contract** тАФ every concrete `Sampler.next_block` returns values strictly in `(0, 1)`. The inverse-cumulative transforms diverge at the endpoints, so any new sampler must respect this; the existing implementations do so by adding 0.5 ULP before normalisation, or by mapping the zero index of a QMC sequence to a non-zero offset.
- **Dimension-aware block shape** тАФ `next_block(n_paths, n_dimensions)` is the *only* sampling entry point. The shape carries the structural dimension that QMC needs; never reshape after the fact. For path engines this also means: pre-decide how many uniforms each path needs (e.g. `n_assets * n_steps` for a basket diffusion), then call `next_block` once per simulation block.
- **Uniforms per normal** тАФ `NormalTransform.uniforms_per_normal` defaults to 1 but is `12` for `CLTTransform`. `NormalSampler.next_block` reads this and asks the underlying sampler for `n_dimensions * uniforms_per_normal` columns. Any new transform that consumes more than one uniform per output must set this class attribute or `NormalSampler` will under-feed it.
- **VolModel signature is permanent** тАФ `diffusion(time, spot, state=None) -> np.ndarray` is the stable contract the path engine calls. The `state` kwarg is plumbed even though current models ignore it; future stochastic-vol siblings (Heston, SABR) will read it without churning the engine call site. Models broadcast a scalar `sigma` to `np.shape(spot)`; spot-dependent models (local vol) will return a true per-path array of the same shape.
- **ATM repricing guarantee** тАФ `BlackTermStructureVol` is constructed so that the MC integral of `sigma_inst^2` over `[0, T_i]` exactly equals `surface.total_variance(T_i, 0.0)` at every pillar. Calendar arbitrage in the source surface (non-monotone ATM `w` in `T`) is rejected at construction with `ValueError` тАФ repairing the surface is the caller's job, not the model's.

## Adding a new uniform sampler

1. Subclass `Sampler` in a new module under `montecarlo/uniform/`. Set the class attribute `is_quasi` correctly (`False` for PRNGs, `True` for low-discrepancy sequences) тАФ the factory uses this flag to enforce the pairing rule and there is no other signal.
2. Implement `next_block`, `reset`, and the `state` property. `reset()` must be idempotent and restore the post-construction state exactly; consumers rely on this for reproducibility.
3. Add the class to `montecarlo/uniform/__init__.py` and re-export it from `montecarlo/__init__.py`'s `__all__`.
4. Add a parameterised test in `tests/test_montecarlo_uniform.py` (shape, range, reproducibility after reset). For PRNGs also add to the KS coverage; for QMC samplers add a discrepancy assertion.

## Adding a new normal transform

1. Subclass `NormalTransform` in a new module under `montecarlo/normal/`. Set `qmc_safe` correctly тАФ `True` only if the transform is a monotone single-uniform-in / single-normal-out inverse-cumulative function. If it consumes more than one uniform per output, set `uniforms_per_normal` accordingly.
2. Implement `transform(uniforms)`. Input shape is `(n_paths, n_dimensions * uniforms_per_normal)`; output shape is `(n_paths, n_dimensions)`. Operate vectorised; element-wise branches use `np.where`.
3. Add the class to `montecarlo/normal/__init__.py` and re-export it from `montecarlo/__init__.py`'s `__all__`.
4. Add an accuracy test in `tests/test_montecarlo_normal.py` against `scipy.special.ndtri` over a grid covering central and tail regions. Add a moments check via the `make_normal_sampler` factory.

## Logging convention

Every module in this package declares `logger = logging.getLogger(__name__)` and uses `%`-style lazy formatting. The factory (`montecarlo.normal.factory`) emits an INFO line per `NormalSampler` construction. Hot paths тАФ the MT19937 refill and the Sobol direction-table build тАФ log DEBUG, guarded by `logger.isEnabledFor(logging.DEBUG)`. New solvers and iterative diagnostics in this package should follow the same INFO-summary + guarded-DEBUG-per-iteration shape used by `credit/bootstrapper.py`.

## Future hooks

These are deliberately staged for subsequent PRs; their interfaces are stubbed or fixed here so adding them later does not churn the existing API:

- **Antithetic variates** (PR 2 of the path-engine roadmap) тАФ an `AntitheticNormalSampler` wrapper that produces `U` and `1 - U`. Refused on a quasi base by the same rule as the Box-Muller / Sobol clash. `EulerLogPathEngine.__init__` already accepts the `antithetic=True` kwarg slot and currently raises `NotImplementedError`.
- **Brownian Bridge** (PR 3) тАФ reorders the `(n_paths, n_steps, n_assets)` normal block so the largest-variance Brownian increments consume the lowest QMC dimensions. `EulerLogPathEngine.__init__` already accepts the `brownian_bridge=True` kwarg slot and currently raises `NotImplementedError`.
- **Correlated basket draws** (PR 4) тАФ a `Correlation` object holding the correlation matrix and cached Cholesky factor; per-step normals are transformed as `Z @ L.T` before entering the step update. `EulerLogPathEngine.__init__` already accepts the `correlation=...` kwarg slot and currently raises `NotImplementedError`. PCA decomposition is deferred тАФ the static-matrix Cholesky form is sufficient for equity-basket autocalls.
- **MRG32k3a substreams** тАФ `LecuyerMRG32k3a1999Sampler.substream(index)` is the stub; the leap-ahead matrix exponentiation will land if and when basket simulations need independent per-asset streams.
- **Stochastic-vol models** тАФ drop into `volatility/stochastic/` as `heston.py`, `sabr.py`, etc. Each subclass implements `diffusion(time, spot, state)` reading `state["variance"]` (Heston) or `state["alpha"]` (SABR). Path engine already plumbs `state` through `VolModel.diffusion`; the model is stateless from the engine's view.
- **Local volatility (Dupire)** тАФ already shipped at `volatility/local_vol.py` (`DupireLocalVol`). The full skew-aware bridge between `market_structures.volatility.VolSurface` and the path engine.
